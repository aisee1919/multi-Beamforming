# 项目架构改进意见

> 当前定位：实验性项目。目标是方便改实验、看结果、复现实验链路；不追求发布级 API、强防御校验或旧版本兼容。

## 已完成

1. 默认入口已支持 CBF、FB、FFT-FISTA，并通过 `--include-damas-fista` 显式接入 DAMAS-FISTA。
2. DAMAS-FISTA 实验逻辑已下沉到 `experiments/damas_fista.py`，脚本和主入口复用同一套函数。
3. 热力图写出已统一到 `ResultWriter.write_heatmap()`。
4. `ExperimentSummary` 已使用 `algorithm_paths` 字典承载算法输出。
5. 旧版函数式兼容入口已移除：
   - `conventional_beamforming()`
   - `functional_beamforming()`
   - `run_cbf_for_planes()`
   - `run_fb_for_planes()`
6. 大部分显式 `ValueError` 参数校验已删除，代码默认输入来自本项目实验配置。

## 仍建议做

### 1. 轻量集成测试

给入口增加一个只检查 CLI 能启动的测试，避免 `src` 包路径或 argparse 改坏：

```bash
.venv-wsl/bin/python main.py --help
.venv-wsl/bin/python scripts/run_damas_fista_single_source.py --help
```

### 2. 多声源实验

当前所有算法主要验证单声源定位。若要比较旁瓣、分辨率和稀疏反卷积效果，下一步应增加多声源 case，而不是继续增加参数校验。

### 3. DAMAS-FISTA 性能

若需要跑 `step=0.01 m`，优先考虑：

- 缓存 PSF 算子。
- 只在局部 ROI 内跑精确 DAMAS-FISTA。
- 对比 FFT-FISTA 近似结果后再决定是否跑全网格。

### 4. 文档收敛

长期保留的文档只需要三类：

- `README.md`：运行方式。
- `docs/项目复盘总结.md`：项目当前状态。
- `changelogs/`：每次变更记录。

需求文档可作为历史记录保留，但不要把历史验收项当成当前架构约束。
