# 项目架构改进意见

> 目标：易读、简洁、低耦合高内聚、易于扩展

---

## 一、删除冗余重导出模块 [高优先]

**现状**：`domain/array.py`、`domain/source.py`、`domain/scan.py` 每个只有 4 行业务代码，把 `array_geometry.py` / `scene.py` 的符号照抄一遍再导出。`domain/__init__.py` 又从这里二次重导出。

```
用户 import → domain/__init__.py → domain/array.py → array_geometry.py
                               → domain/source.py → scene.py
                               → domain/scan.py  → scene.py
```

**问题**：三层间接引用，定位源码要跳转 3 次。3 个文件 18 行零功能代码。

**改进**：删除这 3 个文件。`domain/__init__.py` 直接从源模块导入：

```python
# domain/__init__.py
from beamforming_sim.array_geometry import MicrophoneArray, SpiralArrayConfig, create_eight_arm_spiral_array
from beamforming_sim.domain.result import BeamformingResult
from beamforming_sim.scene import AcousticSource, ScanPlane, SourceModel, create_default_sources, create_scan_planes
```

**收益**：-3 文件，-18 行，导入链路缩短 1 层。

---

## 二、收窄 `algorithms/__init__.py` 导出 [高优先]

**现状**：导出 13 个符号，包含所有内部原语（`cbf_power_from_csm`、`quadratic_power_from_steering`、`steering_matrix`、`point_chunks`、`csm_power_eig`、`fb_power_from_transformed_csm`、`normalize_power`）。

**问题**：外部调用方（test_cbf、test_fb）只能通过 `beamforming.py` 兼容层访问这些私有符号。如果直接通过 `algorithms/__init__.py` 暴露，就失去了封装。实际上只有 `beamforming.py` 需要这些内部原语，其他模块不需要。

**改进**：`algorithms/__init__.py` 只导出公开 API —— 3 个 Beamformer 类 + Beamformer Protocol：

```python
# algorithms/__init__.py
from beamforming_sim.algorithms.base import Beamformer
from beamforming_sim.algorithms.cbf import ConventionalBeamformer
from beamforming_sim.algorithms.fb import FunctionalBeamformer
from beamforming_sim.algorithms.fft_fista import FFTFISTABeamformer

__all__ = ["Beamformer", "ConventionalBeamformer", "FunctionalBeamformer", "FFTFISTABeamformer"]
```

`beamforming.py` 直接从子模块导入原语（它已经在 `cbf.py` / `fb.py` 内部这样做）。

**收益**：公开 API 从 13 个降到 4 个，需要新增算法时只新增一个类。

---

## 三、合并 CBF/FB 共享 `run()` 流程 [中优先]

**现状**：`ConventionalBeamformer.run()` 和 `FunctionalBeamformer.run()` 有完全相同的 3 行：
```python
validate_beamforming_inputs(array, signals, ...)
csm = compute_cross_spectral_matrix(signals, ...)
return self.run_from_csm(array, plane, csm, ...)
```

**问题**：将来新增算法 D、E、F 时，每个都要复制这 3 行。

**改进**：在 `base.py` 中提供一个 mixin 类：

```python
# algorithms/base.py
class CsmBasedBeamformer:
    """任何基于 CSM 的波束形成器共用的 signal→CSM→result 流程。"""

    def run_from_signals(
        self: Beamformer,
        array: MicrophoneArray,
        plane: ScanPlane,
        signals: np.ndarray,
        sampling_rate_hz: float,
        frequency_hz: float,
        sound_speed_m_s: float = 343.0,
        block_size: int | None = None,
        overlap: float = 0.5,
    ) -> BeamformingResult:
        validate_beamforming_inputs(array, signals, sampling_rate_hz, frequency_hz, sound_speed_m_s)
        csm = compute_cross_spectral_matrix(signals, sampling_rate_hz, frequency_hz, block_size, overlap)
        return self.run_from_csm(array, plane, csm, frequency_hz, sound_speed_m_s)
```

然后 CBF 和 FB 直接继承：
```python
@dataclass(frozen=True)
class ConventionalBeamformer(CsmBasedBeamformer):
    def run(self, ...):  # 委托给 run_from_signals
        return self.run_from_signals(...)
```

**收益**：减少 ~20 行重复，新算法只需实现 `run_from_csm()` 即可自动获得完整 `run()` 流程。

## 四、简化 `beamforming.py` 兼容层 [中优先]

**现状**：122 行，8 个函数 + 4 个别名。私有别名 `_steering_matrix = steering_matrix` 等仅为了让 test_fb.py 的导入路径不报错。

```python
_steering_matrix = steering_matrix
_normalize_energy = normalize_power
_csm_power_eig = csm_power_eig
_validate_signal_inputs = validate_signal_inputs
```

**问题**：别名引入隐蔽耦合——改底层函数名时忘了改别名会导致静默失效。这些别名只被旧的 test_fb.py 使用。

**改进**：直接在 test_fb.py 改导入路径（只需 1 行改动），然后删除全部别名。`_normalize_energy`、`_cbf_from_csm`、`_fb_from_csm` 均可在代码审查时确认无外部引用后移除。

```python
# test_fb.py: 改
from beamforming_sim.beamforming import _csm_power_eig
# 为
from beamforming_sim.algorithms.fb import csm_power_eig
```

**收益**：-20 行别名，兼容层从 122 行精简到 ~100 行。

## 五、合并 `writer.py` 写热力图方法 [中优先]

**现状**：`write_cbf_heatmap`、`write_fb_heatmap`、`write_fft_fista_heatmap` 三个方法结构完全相同：
```python
def write_xxx_heatmap(case, result, ...):
    path = self.output_dir / "xxx" / f"xxx_{case.index:02d}_z_{case.z_m}..."
    return plot_energy_heatmap(case.plane, result.raw_power, path, title=..., tick_step_m=self.tick_step_m)
```

**问题**：每新增一个算法要复制 10 行路径拼装 + title 拼装。现有方法 80% 重复。

**改进**：用一个通用方法 + `BeamformingResult` 自带的信息驱动路径：

```python
def write_heatmap(self, case: SourceCase, result: BeamformingResult, subdir: str) -> Path:
    stem = result.algorithm.lower().replace("-", "_")
    if "nu" in (result.metadata or {}):
        suffix = f"_nu{result.metadata['nu']:02d}"
    else:
        suffix = f"_source_{case.index:02d}_z_{case.z_m:.1f}m_x_{case.x_m:+.1f}_y_{case.y_m:+.1f}"
    path = self.output_dir / subdir / f"{stem}{suffix}.png"
    return plot_energy_heatmap(case.plane, result.raw_power, path,
                               title=f"{result.algorithm}(z={case.z_m:g}m, ...)",
                               tick_step_m=self.tick_step_m)
```

**收益**：3 个方法合并为 1 个，未来新算法自动适配。

## 六、`ExperimentSummary` 使用字典替代硬编码字段 [低优先]

**现状**：每加一种算法，`ExperimentSummary` 就加一个 `xxx_paths: list[Path]` 字段。

**问题**：`main.py` 和 `runner.py` 对每种算法都要写一段专门的打印/收集代码。扩展算法 = 改 3 个地方。

**改进**：用字典统一管理：
```python
@dataclass(frozen=True)
class ExperimentSummary:
    ...
    algorithm_paths: dict[str, list[Path]] = field(default_factory=dict)

# runner.py:
summary.algorithm_paths["CBF"] = cbf_paths
summary.algorithm_paths["FB"] = fb_paths
summary.algorithm_paths["FFT-FISTA"] = fft_fista_paths

# main.py:
for algo, paths in summary.algorithm_paths.items():
    print(f"\n{algo} heatmaps ({len(paths)}):")
    for p in paths: print(f"  {p}")
```

**收益**：新增算法时 runner 加 2 行，main.py **不用改**。

## 七、`runner.py` 算法流程用注册表替代硬编码 [低优先]

**现状**：`run()` 方法中 CBF、FB、FFT-FISTA 的调用各自硬编码：
```python
cbf_result = cbf.run_from_csm(...)
cbf_paths.append(writer.write_cbf_heatmap(...))
fb_paths.extend(self._run_fb_cases(...))
fft_fista_result = fft_fista.run_from_cbf_map(cbf_result, ...)
fft_fista_paths.append(writer.write_fft_fista_heatmap(...))
```

**问题**：每种算法的插入位置、依赖关系不同（FB 依赖 CSM，FFT-FISTA 依赖 CBF 结果），扩展时容易破坏流程。

**改进**：定义算法 pipeline 步骤：
```python
PIPELINE = [
    ("CBF", ConventionalBeamformer(), needs_cbf_result=False),
    ("FB", None, needs_cbf_result=False),           # 由 _run_fb_cases 处理
    ("FFT-FISTA", FFTFISTABeamformer(), needs_cbf_result=True),
]
```

然后遍历执行。这样新增算法只需在列表加一行。

## 九、其他小改进

### 9.1 放松 `arms==8` 硬编码
`array_geometry.py:_validate_config` 第 73 行 `if config.arms != 8: raise ValueError`。改为 `if config.arms < 2: raise ValueError`。当前即使 arms=4 也能正确生成 4 臂螺旋，只是注释说"参照八臂实验"。保留默认值 8 即可，不阻止用户传入其他值。

### 9.2 添加 `pyproject.toml` 类型检查配置
```toml
[tool.mypy]
python_version = "3.11"
strict = false
```

### 9.3 `_inclusive_grid` 增加 `atol` 参数
当前第 106 行 `np.isclose(values[-1], stop)` 使用默认容差，对某些极端步长/范围组合可能判错。加 `atol=1e-9` 显式控制。

### 9.4 `_fista_deconvolution` 溢出保护
当前迭代中 `Ay`、`grad`、`x_next` 无上限裁剪。在极端 lambda 下可能溢出。加 `x_next = np.clip(x_next, 0, np.max(dirty_map) * 2)` 防止发散。

## 十、改进优先级总结

| 序号 | 改进 | 影响 | 工作量 |
|------|------|------|--------|
| 1 | 删除 `domain/{array,source,scan}.py` | 减 3 文件 18 行 | 10 min |
| 2 | 收窄 `algorithms/__init__.py` 导出 | API 从 13→4 符号 | 5 min |
| 3 | 移除 `seaborn` | 零影响清理 | 2 min |
| 4 | 合并 `writer.py` 写热力图 | 3→1 方法 | 15 min |
| 5 | CBF/FB 共享 `run()` 流程 | -20 行重复 | 15 min |
| 6 | 清理 `beamforming.py` 别名 | -20 行 | 10 min |
| 7 | `ExperimentSummary` 用字典 | 扩展不改 main.py | 15 min |
| 8 | `runner.py` pipeline 注册表 | 新算法加 1 行 | 30 min |
| 9 | 放松 `arms==8`、atol、溢出保护 | 健壮性 | 10 min |
| 10 | 添加 mypy 配置 | 类型安全 | 5 min |

**推荐执行顺序**：1、2、3 先做（清理型，改动小收益大），4、5、6 第二批（结构优化），7、8 最后做（设计升级，需要更多测试）。
