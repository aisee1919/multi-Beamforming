# 多算法波束形成仿真项目 — 复盘总结

> **项目**: beamforming-sim v0.1.0  
> **日期**: 2026-05-06  
> **环境**: Python 3.11+, NumPy, Matplotlib, SciPy  
> **测试**: 30 项，全部通过  
> **提交**: 3 次（初始 → FB → FFT-FISTA+架构重构）

---

## 一、项目做了什么

本项目是一个 **声学波束形成多算法仿真框架**。在自由场近场条件下，对点声源进行定位成像，支持三种波束形成算法的对比分析：

| 算法 | 公式 | 特点 |
|------|------|------|
| **CBF**（常规波束形成） | P = w^H · C · w | 基线算法，稳健但空间分辨率有限 |
| **FB**（函数波束形成） | B = [w^H · C^(1/ν) · w]^ν | 对 CSM 做特征分解后降幂再升幂，ν 越大主瓣越窄 |
| **FFT-FISTA**（反卷积成像） | min 0.5‖PSF∗x−b‖² + λ‖x‖₁, x≥0 | 在 CBF dirty map 上做稀疏反卷积，空间分辨率最高 |

核心场景：64 阵元八臂螺旋阵列 → 9 个 25 kHz 点声源（3 距离 × 3 横向位置）→ 每个声源单独成像 → 生成 54 张 dB 热力图。

## 二、如何做的

### 2.1 物理建模链路

```
点声源 → 自由场传播（1/r 衰减 + 时延） → 64 路麦克风信号 → CSM → 算法 → 能量图
```

1. **阵列几何**：`SpiralArrayConfig` → `create_eight_arm_spiral_array()` 在孔径 0.15 m 圆内生成 8 臂 × 8 阵元螺旋排列
2. **信号仿真**：对每个声源，计算到各麦克风的传播时延和 1/r 幅值衰减，叠加正弦波，可选加 AWGN
3. **频谱分析**：汉宁窗分块 FFT → 目标频率 25 kHz 处提取复幅度 → 外积平均 → Hermitian 对称 CSM
4. **算法计算**：基于 CSM 和导向矩阵输出扫描平面上的能量分布
5. **可视化**：dB 相对峰值热力图 (jet colormap, -50 dB floor)

### 2.2 三个算法的核心差异

| | CBF | FB | FFT-FISTA |
|---|---|---|---|
| **输入** | 信号 → CSM | 信号 → CSM | CBF 的 `BeamformingResult` |
| **核心运算** | einsum 二次型 | 特征分解 + einsum | FFT 卷积/反卷积 + FISTA 迭代 |
| **复杂度** | O(N_scan × N_mic) | O(N_mic³ + N_scan × N_mic) | O(N_iter × N_scan × log N_scan) |
| **输出特性** | 平滑宽主瓣 | 窄主瓣 (ν↑) | 稀疏重构（主瓣最窄） |

### 2.3 FFT-FISTA 关键实现细节

- **PSF 构造**：取扫描平面中心点，构造 rank-1 CSM，跑 CBF 得到点扩散函数。**平移不变近似**——实际 PSF 随空间位置变化。
- **卷积对齐**：PSF 需要 `ifftshift` 后再 FFT，否则卷积结果偏移半个网格（峰值定位误差 ~0.57 m → 0.00 m 的核心修复）。
- **Lipschitz 常数**：L = max(|FFT(ifftshift(PSF))|²)，从频域直接估计。
- **Nesterov 加速**：标准 FISTA 动量项，实测 200 次迭代可收敛。

## 三、做得好不好

### 3.1 优点

| 维度 | 评价 |
|------|------|
| **算法正确性** | CBF 峰值定位准确；FB ν=1 严格等于 CBF；FFT-FISTA 峰值误差 0.000 m；FB 主瓣宽度随 ν 单调递减 |
| **测试覆盖** | 30 项测试：6 仿真 + 2 可视化 + 4 CBF + 5 FB + 8 FFT-FISTA + 6 架构；覆盖核心正确性、接口契约、参数校验 |
| **API 设计** | `Beamformer` Protocol + `BeamformingResult` 冻结数据类，类型安全、可扩展 |
| **向后兼容** | `beamforming.py` 门面层保证旧调用方零修改 |
| **CSM 复用** | CSM 与扫描平面无关，一次计算多平面复用 |
| **内存控制** | `point_chunks` 分块避免大网格 OOM；FFT 替代空间域卷积 |

### 3.2 待改进

| 维度 | 问题 |
|------|------|
| **冗余模块** | `domain/array.py`、`source.py`、`scan.py` 是空垫片，仅重新导出，增加 3 文件 18 行无功能代码 |
| **未使用依赖** | `seaborn` 在 pyproject.toml 声明但源码中未引用 |
| **代码重复** | CBF 和 FB 的 `run()` 方法流程几乎相同，未抽取基类 |
| **测试缺口** | 可视化函数无直接测试；多声源并发场景无测试；边缘情况（极短信号、高噪声）未覆盖 |
| **硬编码限制** | `_validate_config` 强制 `arms == 8`，拒绝非八臂阵列 |
| **类型检查** | 无 mypy/pyright 配置，ndarray 形状仅靠运行时验证 |
| **文档** | 无 README、无 LICENSE、无 API 参考（本文档即补充） |

## 四、软件设计方案

### 4.1 架构原则

1. **关注点分离**：算法、频谱、领域、实验四层隔离
2. **协议优于继承**：Python `Protocol` 定义 `Beamformer` 接口，结构子类型化而非 ABC 继承
3. **不可变数据流**：冻结数据类 (`frozen=True`) 保证结果对象不可变，`MappingProxyType` 保护元数据
4. **门面兼容**：旧函数 API 通过门面层委托给新类，实现平滑迁移
5. **依赖注入**：`ExperimentConfig` 注入到 `BeamformingExperiment`；CSM 可注入 `run_from_csm()`

### 4.2 设计模式

| 模式 | 位置 | 作用 |
|------|------|------|
| **门面** | `beamforming.py`, `__init__.py` | 简化外部访问，隔离内部变化 |
| **策略/协议** | `algorithms/base.py:Beamformer` | 算法可互换 |
| **数据类** | 所有 `@dataclass(frozen=True)` | 减少样板，语义清晰 |
| **依赖注入** | `run_from_csm(csm=...)` | 测试可复用 CSM |
| **分块迭代器** | `point_chunks()` | 大网格内存控制 |

### 4.3 关键设计决策

1. **raw_power 不做归一化**：算法层保存线性原始功率，归一化推迟到 `BeamformingResult.normalized_power()` 和 `visualization.py`。这避免了 FFT-FISTA 在 dB 域做优化。
2. **PSF 平移不变近似**：工程折中。完全平移变化的 PSF 需要在每个扫描点单独构造，计算量翻倍。中心 PSF 近似在 ±0.6 m 范围内可接受。
3. **CSM 一次计算复用**：`compute_cross_spectral_matrix` 独立于扫描平面，`run_cbf_for_planes`/`run_fb_for_planes` 共享同一 CSM。
4. **兼容层保留私有别名**：`_steering_matrix = steering_matrix` 等别名使旧测试无需改导入路径即可通过。
